## Übersicht

Um Compliance-Ergebnisse mit realen Lieferungen zu verknüpfen, müssen Sie Ihre Bestellungen in Polygon One importieren. Jede Bestellung erfasst den Lieferanten, die Artikel und die Mengen.

<Frame caption="Bestellungsübersicht">
  <img src="/images/helpcenter/dds-bestellungen/bestellungen-liste.png" />
</Frame>

## Einkauf vs. Verkauf

<Info>
In Polygon One wird zwischen zwei Bestelltypen unterschieden:

- **Einkauf** (Purchase) — Eingehende Waren von Ihren Lieferanten
- **Verkauf** (Sale) — Ausgehende Waren an Ihre Kunden (Aufträge)
</Info>

## Import-Methoden

<Tabs>
  <Tab title="CSV Import">
    **Ideal für die Verarbeitung großer Datenmengen.**

    1. Navigieren Sie zum Bereich **Bestellungen**.
    2. Klicken Sie auf **Importieren** und laden Sie Ihre CSV-Datei hoch.
    4. Bestätigen Sie den Import.

    <Frame caption="CSV-Import mit Spaltenzuordnung für Bestellungen">
      <img src="/images/helpcenter/dds-bestellungen/bestellungen-import.png" />
    </Frame>
  </Tab>
  <Tab title="Manuelle Erstellung">
    **Für einzelne Bestellungen.**

    1. Klicken Sie auf **Erstellen** im Bestellungsbereich.
    2. Wählen Sie den **Lieferanten** aus.
    3. Geben Sie **Bestellnummer** und **Bestelldatum** ein.
    4. Wählen Sie den **Bestelltyp** (Einkauf/Verkauf).
    5. Fügen Sie **Positionen** hinzu (Artikel + Volumen in kg).
    6. Speichern Sie die Bestellung.

    <Frame caption="Bestellung manuell erstellen">
      <img src="/images/helpcenter/dds-bestellungen/bestellungen-erstellen.png" />
    </Frame>
  </Tab>
  <Tab title="REST API">
    **Für die automatisierte Synchronisierung.**

    Synchronisieren Sie Bestellungen direkt aus Ihrem ERP-System über die API.
    Siehe [API-Referenz](/api-reference/endpoint/orders/create) für Details.
  </Tab>
</Tabs>

## CSV-Felder

| Feld | Pflicht | Beschreibung |
|------|---------|-------------|
| **Bestellnummer** | Ja | Eindeutige Bestellnummer |
| **Bestelltyp** | Ja | `purchase` (Einkauf) oder `sale` (Verkauf) |
| **Lieferant** | Ja | Lieferantenname |
| **Artikelnummer** | Ja | Interne Artikelnummer |
| **Menge** | Ja | Bestellmenge (in kg) |
| **Bestelldatum** | Nein | Datum der Bestellung |
| **Externe Lieferanten-ID** | Nein | Hat Vorrang vor dem Lieferantennamen beim Abgleich |
| **Lieferanten-E-Mail** | Nein | Ergänzt automatisch erstellte Lieferanten |
| **Lieferantenland** | Nein | Ergänzt automatisch erstellte Lieferanten |
| **Lieferantenadresse** | Nein | Ergänzt automatisch erstellte Lieferanten |
| **HS-Code** | Nein | Erforderlich zur Artikelerstellung, wenn Artikel noch nicht existiert |
| **DDS-Referenznummer** | Nein | Externe DDS-Referenz (für Durchreichung vorgelagerter DDS) |
| **DDS-Prüfnummer** | Nein | Externe DDS-Verifizierungsnummer |

<Info>
Beim Bestellimport können Lieferanten und Artikel automatisch angelegt werden, wenn sie noch nicht existieren. Das System gleicht anhand von Name bzw. externer ID ab.
</Info>

## Spaltenzuordnung konfigurieren

Unter **Einstellungen → CSV-Importe → Bestellungen** können Sie Ihre eigenen Spaltennamen dauerhaft den Polygon One Feldern zuordnen. So müssen Sie die Zuordnung nicht bei jedem Import erneut vornehmen.

## EUDR-Relevanz

Das System nutzt die **HS-Codes** der Artikel in Ihren Bestellungen, um automatisch zu bestimmen, ob eine Bestellung **EUDR-relevant** ist.

<Note>
Nur Bestellungen mit EUDR-relevanten Artikeln erscheinen in den DDS-Zuweisungsfiltern.
</Note>

<Tip>
Wenn Sie regelmäßig Bestellungen importieren, empfiehlt sich die Integration über die REST API für eine vollautomatische Synchronisierung.
</Tip>
