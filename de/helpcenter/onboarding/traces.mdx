## Was ist EU TRACES?

EU TRACES NT (Trade Control and Expert System) ist das offizielle System der Europäischen Union für die Einreichung von Due Diligence Statements (DDS). Um DDS über Polygon One einreichen zu können, benötigen Sie einen aktiven EU TRACES Account mit einem **Web Service Access Authentication Key**.

## Wann brauche ich TRACES?

<Info>
Sie benötigen EU TRACES Zugangsdaten, sobald Sie Due Diligence Statements erstellen und an die EU übermitteln möchten. Für die vorgelagerten Schritte (Datenhaltung, Analyse, Datensammlung) ist TRACES **nicht** erforderlich.
Außerdem haben sie auch die Möglichkeit DDS Vorlagen zu erstellen und nicht an direkt an Traces zu übermitteln.
</Info>

## Vereinfacht erklärt

Bevor Polygon One DDS an die EU übermitteln kann, müssen Sie Ihr Unternehmen im offiziellen EU-System (TRACES NT) registrieren und einen **Web Service Authentication Key** erhalten. Der Prozess umfasst:

1. EU Login Account erstellen
2. Ihr Unternehmen als Operator registrieren
3. Sich dem Operator als Benutzer zuweisen
4. Den Web Service Authentication Key per E-Mail beantragen
5. Den Key in Polygon One hinterlegen

---

## Schritt 1: Offizielle Dokumentation lesen (optional)

Lesen Sie die offizielle EUDR API-Dokumentation, um die technischen Anforderungen zu verstehen.

<Note>
Die offizielle EUDR-Dokumentation ist sehr technisch. Wenn Sie zum ersten Mal mit dem EUDR-System arbeiten, können Sie diesen Schritt überspringen und später darauf zurückkommen. Polygon One bietet eine benutzerfreundliche Oberfläche für die meisten Vorgänge.
</Note>

## Schritt 3: Operator-Entity in TRACES NT erstellen

Im EUDR Portal müssen Sie Ihr Unternehmen als **Operator** mit spezifischen Aktivitäten und vollständigen Firmendaten registrieren. Dieser Schritt bestimmt Ihre Rolle in der Lieferkette.

<Warning>
- Die Operator-Entity muss von einem Administrator **validiert** werden, bevor Benutzer zugewiesen werden können
- Alle Angaben müssen mit Ihren offiziellen Handelsregisterdaten übereinstimmen
- Doppelte Operator-Einträge werden nicht validiert
</Warning>

## Schritt 4: Benutzerrolle dem Operator zuweisen

Nach dem Erstellen der Operator-Entity müssen Sie Ihren Benutzeraccount dem Operator zuweisen:

<Steps>
  <Step title="Profil bearbeiten">
    Navigieren Sie zu **"Edit your profile"** im EUDR Portal.
  </Step>
  <Step title="Neue Rolle anfordern">
    Wählen Sie **"Request new role"** und dann **"Operator"**.
  </Step>
  <Step title="Operator suchen">
    Suchen Sie Ihre Operator-Entity nach Land und EUDR-Sektion.
  </Step>
  <Step title="Autorisierung senden">
    Senden Sie die Autorisierungsanfrage, um Benutzer des Operators zu werden.
  </Step>
</Steps>

<Warning>
Die Operator-Entity muss durch einen Administrator **validiert** sein, bevor Sie den nächsten Schritt (Web Service Authentication Key beantragen) durchführen können. Die Validierung kann **1–3 Werktage** dauern.
</Warning>

## Schritt 5: Web Service Authentication Key beantragen

Nachdem alle vorherigen Schritte abgeschlossen sind, werden Sie feststellen, dass das Feld **Web Service Client Authentication Key** in Ihrem EUDR Portal leer ist. Das ist normal — dieser Key wird **nicht automatisch generiert**, sondern muss per E-Mail beantragt werden.

Senden Sie eine E-Mail an **SANTE-TRACES@ec.europa.eu** mit folgender Vorlage:

### E-Mail-Vorlage

```
An: SANTE-TRACES@ec.europa.eu
Betreff: Request for Web Services Authentication Key - [Ihr Firmenname]

Dear EUDR Support Team,

I am writing to request Web Services Authentication Key for the EUDR
Information System (TRACES NT) to enable API integration with your system.

Company Information:
- Company Name: [Ihr Firmenname]
- VAT ID: [Ihre Umsatzsteuer-ID]
- EUDR Portal Username: [Ihr EUDR Benutzername]

I need Web Services Authentication Key for the following environment:
- Acceptance Environment (for testing and development)

Please provide the Web Service Access Authentication Key that will allow
our system to integrate with the EUDR API.

Thank you for your assistance.

Best regards,
[Ihr Name]
[Ihre Kontaktinformationen]
```

<Info>
**Bearbeitungszeit:** 1–2 Werktage. Sie erhalten den Key per E-Mail nach Genehmigung.
</Info>

## Schritt 6: Konfiguration überprüfen

Sobald Sie Ihren Web Service Authentication Key erhalten haben, prüfen Sie im EUDR Portal, ob Ihr Unternehmen korrekt konfiguriert ist. Das Feld **Web Service Access** sollte nun befüllt sein.

<Frame caption="TRACES Portal mit befülltem Web Service Access Feld">
  <img src="/images/helpcenter/onboarding/traces-webservice-key.png" />
</Frame>

## Schritt 7: Zugangsdaten in Polygon One hinterlegen

Hinterlegen Sie Ihre EUDR-Zugangsdaten in Polygon One unter **Einstellungen → Profil**:

<Frame caption="TRACES-Zugangsdaten in Polygon One Einstellungen">
  <img src="/images/helpcenter/onboarding/traces-polygon-einstellungen.png" />
</Frame>

| Feld | Beschreibung |
|------|-------------|
| **EORI Nummer** | Ihre EORI Nummer. Diese wird fest benötigt um DDS einzureichen |
| **TRACES Benutzername** | Ihr registrierter EU TRACES Benutzername (Format: `n00ixxx`) |
| **Traces Web Service Access Token** | Key für die EUDR Umgebung (TEST oder PRODUKTIV) |

Klicken Sie auf **"Speichern"**, um die Konfiguration abzuschließen.

<Warning>
**Sicherheitshinweise:**
- Bewahren Sie Ihre EUDR-Zugangsdaten sicher auf und teilen Sie diese nicht öffentlich
- Verwenden Sie Acceptance-Zugangsdaten nur für Tests und Entwicklung
- Produktions-Zugangsdaten sollten nur für den Echtbetrieb verwendet werden
</Warning>
